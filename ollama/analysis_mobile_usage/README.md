# スマホ料金診断アプリ(Flask)

## できること
- 入力：データGB、5分以内通話回数
- 出力：povo / LINEMO の見積もり月額、recommended

## 仕様
- 入力項目: データGB、5分かけ放題の回数
- 出力フォーマット: povo、LINEMOの見積もり月額、レコメンド

## 設計上の判断
- 本アプリでは、入力項目を「データ使用量」と「5分以内の発信回数」に限定し、料金診断ロジックの分岐構造を明確にすることを優先した
- 発信分数ベースの診断は、通話時間の合計計算や複数オプション判定が必要となり、条件分岐と状態管理が増えるため、本プロジェクトの主目的である「料金判定ロジックの可読性」を損なうと判断した
- そのため本段階では、通話オプションの有無のみを判定対象とし、拡張可能な構造を保ったまま実装している

## 使い方（再現手順）

- ブラウザでアクセス
- テスト実行

## 制限事項(未対応)
- 通話分数ベースの料金計算（通話時間の合計を入力として受け取っていないため未対応）
- 24時間かけ放題の判定（通話分数・通話パターンを前提とするため未対応）
- 時間帯・曜日による料金変動の考慮（時間帯・曜日の入力を受け取っていないため未対応）
## 前提

- 本アプリでは、通話に関する判定を「5分以内の発信回数」を入力値として扱う
- 実務の料金診断では発信分数や通話時間帯なども考慮されるが、本プロジェクトでは以下を優先した
  - 入力項目を最小限に抑え、UIとロジックの接続を明確にすること
  - Python側の料金計算ロジックをシンプルに保ち、仕様判断がコード上で追いやすい構成にすること
- そのため、通話に関しては「5分かけ放題を付与するかどうか」の判断に限定している
