# 設計メモ
## 予測内容
来年度のSIM申込件数の需要予測
月次に分解可能の年間需要予測

## 特徴量
- 日付(datetime)
- 曜日(0~6)
- 月別(1~12)
- 過去n日・nヶ月の申込件数（ラグ特徴量）

## 正解ラベル
- 前年のSIM申込件数の年間データ(CSV)

## 使用目的
今年度のSIM申込の目標件数設定の目安

## 構成 編集必要!!!!!!!!!
- images/
- csv/
- docs/
- analysis_sim_apllications.ipynb

### 設計方針（Config管理）
本プロジェクトでは、データパスやモデル保存先などの設定値を
各ファイルに分散させず、`config.py` に集約しています。

これにより、
- train / predict 間でのパス不整合防止
- ディレクトリ構成変更時の修正範囲最小化
- モデル追加・バージョン管理の容易化

# 分析結果
- 月次データをzスコアで標準化した結果、3月は平均から大きく正の乖離を示し、5〜6月は負の乖離が確認できた。
これは新生活・決算期による需要集中と、その反動による閑散期を反映していると考えられる。
- 月次合計では3月に一時的なピークが見られるが、3ヶ月移動平均では4月以降に下落し、9月から回復トレンドに転じていることが分かる。
- 閑散期にはイベントやキャンペーンに予算を多く配分せず、お店に末永く来てくれるお客様が増えてもらえるような行動を促す。
お客様が少ない、1組の接客に使用できる時間が繁忙期より、多く取れる為、購入後のアフターをしっかり行う。
- 大きく需要が増す1月ごろから競合他社に負けないようにイベントやキャンペーンに予算を多く使い、たくさん契約を取る、それを円滑に回していく、クレームやトラブルを起こさないことに重点を置く。
- 本分析は、年間の販促予算配分および人員配置計画を検討するための基礎資料として活用できる。
