## データ設計の考え方

本データは、携帯販売の現場で実際に発生する質問をもとに、
自分自身で作成したラベル付きデータである。
特に MNP と料金の境界が曖昧な質問を意図的に含めており、
実運用を想定した分類タスクとなっている。

### 1. データ作成の背景

本プロジェクトで作成するデータは、
携帯販売の業務の中で、実際にお客様から聞かれる質問の内容を元に作成したデータセットである。

一般的な公開データセットや自動生成データではなく、

- 実店舗での接客経験
- 顧客の言い回し・曖昧さ・感情
- 店舗スタッフが瞬時に判断すべき「質問の意図」

を反映した実運用を元にデータ設計を重視している。


### 2. タスク定義

本データは以下の分類タスクを目的としている。
- 入力：顧客からの質問文(text)
- 出力：質問の目的カテゴリ（label）

分類ラベル
- 解約・MNP
- 料金

この分類は、
「質問を聞いた瞬間に、どの説明フローに進むか」
という店舗業務の意思決定をそのままモデル化したものである。

### 3. ラベル設計の方針

本データでは、あえて境界が曖昧な質問も含めている。

例:
```text
先日MNPの契約をここでしたが聞いていた料金と違う
```

この質問は
- MNPカテゴリの文脈を含む
- しかし顧客の主な不満・関心は「料金」

そのためラベルは 料金 としている。

これは
「単語一致」ではなく
“質問の主目的を理解できているか”
をモデルに求めるための設計である


### 4. なぜデータを自作したのか

本データを自作した理由は以下の通り。
- LLM生成データは文が整いすぎ、実務では不自然になりやすい
- 実際の顧客質問は、文法が崩れ、背景情報が不足していることが多い
- 分類タスクの難しさは「曖昧さ」にあるため

そのため、本プロジェクトでは
現場知識を持つ自分自身がお客様からの実際の質問を元に作成したデータ
を使用している。


### 5. データサイズについて

本データセットは件数としては小規模である。

これは以下を意図している。
- 小規模データでもベースラインモデルを正しく評価できるか
- 誤分類結果を人間がすべて確認できる規模にする
- 「精度」より「設計・分析プロセス」を重視する


### 6. 今後の拡張想定

本データ設計は、以下の拡張を想定している。
- ラベル追加（例：操作, 故障, プラン相談）
- マルチラベル分類への拡張
- FAQ自動振り分け・有人対応支援への応用
- LLMとの併用（分類 → 回答生成）

### 7. 評価
日本語FAQ文書を対象に char n-gram + Logistic Regression による
テキスト分類モデルを構築した。

特に「解約・MNP」カテゴリでは高い再現率を示し、
一次受付・振り分け用途として十分な性能を確認できた。

「料金」と「端末・機種変更」は意味的に近く、
混同が発生する傾向が見られたが、
これは実務上も想定される挙動であり、
今後のデータ拡充により改善可能と判断した。
